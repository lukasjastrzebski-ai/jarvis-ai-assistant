# Success Metrics

**Stage:** 1
**Status:** Complete
**Date:** 2026-01-15

---

## North Star Metric

**Time Saved Per Week**

Measured via:
- Actions executed by Jarvis
- Triage reduction (items processed vs. time spent)
- User self-reported estimates (survey)

Target: **5+ hours/week** within 30 days of activation

---

## Primary Metrics

### 1. Activation
**Definition:** User completes first successful action (send email, schedule meeting, create task)

| Metric | Target | Rationale |
|--------|--------|-----------|
| Time to first action | <5 minutes | Quick value demonstration |
| First-day actions | 3+ | Proves immediate utility |
| First-week actions | 20+ | Habit formation indicator |

### 2. Engagement
**Definition:** Regular, valuable use of Jarvis

| Metric | Target | Rationale |
|--------|--------|-----------|
| DAU/MAU ratio | >60% | High daily relevance |
| Actions per day | 10+ | Core usage |
| Voice vs. visual ratio | 40%+ voice | Voice-first adoption |
| Inbox zero rate | >70% daily | Unified inbox success |

### 3. Retention
**Definition:** Users continue paying and using

| Metric | Target | Rationale |
|--------|--------|-----------|
| Month 1 retention | >80% | Product-market fit |
| Month 3 retention | >70% | Sustained value |
| Month 6 retention | >60% | Long-term fit |
| Monthly churn | <5% | Healthy business |

### 4. Trust Progression
**Definition:** Users grant more autonomy over time

| Metric | Target | Rationale |
|--------|--------|-----------|
| Assist → Operator | 50% by week 2 | Trust building |
| Operator → Autopilot | 30% by month 2 | Advanced trust |
| Permission grants | Increasing trend | Expanding scope |

## Secondary Metrics

### Productivity Impact
| Metric | Target | Measurement |
|--------|--------|-------------|
| Open loops reduced | 30%/week | Waiting-on list shrinkage |
| Response latency | <4 hours | Key stakeholder emails |
| Context switches | <20/day | App switching reduction |
| Weekly review completion | >80% | Process adoption |

### Quality Metrics
| Metric | Target | Measurement |
|--------|--------|-------------|
| Draft acceptance rate | >90% | Drafts sent without edit |
| Scheduling accuracy | >95% | No conflicts created |
| Action reversal rate | <2% | Wrong actions taken |
| Error recovery time | <1 minute | When mistakes happen |

---

## Anti-Metrics (Things to NOT Optimize)

| Metric | Why NOT |
|--------|---------|
| Time in app | We want less time, not more |
| Features used | Quality over quantity |
| Notifications sent | Less is more |
| Messages exchanged | Actions > chat |

---

## Measurement Plan

### Automated Tracking
- Action counts (type, success, time)
- Session duration and frequency
- Feature adoption funnel
- Permission level changes
- Error rates and types

### User Research
- Weekly NPS survey (sample)
- Monthly deep interviews (5-10 users)
- Quarterly satisfaction survey (all)
- Churn exit interviews

### Cohort Analysis
- Activation cohorts (by first-action type)
- Permission cohorts (Assist/Operator/Autopilot)
- Use-case cohorts (email-heavy vs. calendar-heavy)
